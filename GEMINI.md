# btch-downloader Project Instructions

This document provides mandatory guidance and instructions for AI agents (including Gemini) working within the **btch-downloader** repository.

## Core Architecture & Mandates

- **Primary Goal:** This project is a client SDK (TypeScript/JavaScript) for downloading media from social platforms.
- **NO SCRAPING:** Never add direct scraping logic to this library. All requests MUST be delegated to the backend service at `backend1.tioo.eu.org`.
- **Language:** All new development must be in **TypeScript** (`.ts` files only).
- **Source Location:** Main source code resides in the `src/` directory.
- **Auto-generated Files:** Never manually edit files in `lib/Browser/` or `dist/` as they are auto-generated by the build pipeline.
- **Node.js Compatibility:** Code must be compatible with Node.js **v20.18.1** or higher.
- **Deprecated Modules:** The `mediafire` and `aio` services are no longer maintained. Do not extend or develop new features for these modules.

## Coding Standards & Response Format

Each platform function must return a `Promise<object>` with the following standard structure:

```typescript
{
  status: boolean,    // true = success
  developer: string,  // Must always be "@prm2.0"
  data: object        // Platform-specific media payload
}
```
*Platform-specific response types should be imported from `btch-downloader/Types`.*

## Workflow for Adding New Platforms

When adding support for a new platform, follow these steps sequentially:

1. **Implement Function:** Create a new file in `src/<platform>.ts` and export `async function <platform>(url: string): Promise<object>`.
2. **Export Module:** Re-export the function from `src/index.ts`.
3. **Define Types:** Add the TypeScript response type to `src/Types/`.
4. **Testing:** Create a new unit test in the `test/` directory (e.g., `test/<platform>.test.ts`).
5. **Documentation:** Update `README.md` with both ESM and CJS usage examples.

## Development Commands

Use the following commands to validate changes:

- **Build:** `npm run build` or `pnpm build` (uses `tsup`).
- **Test:** `npm test` or `pnpm test`.
- **Lint:** `npm run lint`.
